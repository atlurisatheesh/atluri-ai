## 2) Visual Density Scoring Framework (0–10)

Use whole-number scores per dimension.

- **Above-the-fold clarity**
  - 0: unclear what product does
  - 5: understandable after reading
  - 10: understood in one glance
- **CTA dominance**
  - 0: competing actions
  - 5: one primary but diluted
  - 10: one obvious action path
- **Section rhythm**
  - 0: uneven and jumpy
  - 5: mostly consistent spacing
  - 10: calm, predictable cadence
- **Whitespace authority**
  - 0: cramped
  - 5: acceptable spacing
  - 10: deliberate negative space drives hierarchy
- **Accent restraint**
  - 0: noisy accents everywhere
  - 5: moderate discipline
  - 10: accent appears only when decision matters
- **Text compression**
  - 0: long copy blocks
  - 5: mixed long/short
  - 10: concise, high-signal lines only
- **Emotional temperature**
  - 0: hype and urgency
  - 5: neutral SaaS tone
  - 10: calm, cold, controlled confidence
- **Cognitive load**
  - 0: too much to parse
  - 5: manageable
  - 10: effortless scanning
- **Conversion friction**
  - 0: unclear next step
  - 5: action is visible but interrupted
  - 10: low-friction path from first view to click
- **Premium positioning**
  - 0: commodity template
  - 5: polished but generic
  - 10: unmistakably high-end restraint
- **Brand distinctiveness**
  - 0: interchangeable with competitors
  - 5: some unique voice
  - 10: instantly ownable personality
- **Narrative flow**
  - 0: random feature order
  - 5: partial arc
  - 10: tight sequence Problem → Outcome → Mechanism → Proof → CTA

### Interpretation Thresholds

- **0–4:** cluttered

- **8–9:** premium
- **10:** category-defining

## 3) 10-Second First Impression Test Checklist

- User should understand: this product helps improve interview performance with clear next steps.
- User should understand: there is one immediate action to start.
- Eye should land first on hero headline, then the primary CTA.
- Eye should then move to outcome headline and 2 bullets.
- Emotional reaction should be: calm, capable, in control.
- Should NOT happen: scanning multiple equal CTAs.
- Should NOT happen: hunting through dense feature explanations.
- Should NOT happen: confusion about who this is for.
- Failure signal: user asks “What does this actually do?” after 10 seconds.
- Failure signal: user cannot describe the next action in one sentence.
